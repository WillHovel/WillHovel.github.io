Overview

A suite of 14 custom Spigot plugins written in Java for a private Minecraft server. The most complex — Manhunt — required real-time position tracking across multiple players, dimension-aware compass updates, and event-driven state management for an indefinite session.

These projects were built for personal use, but they required the same engineering discipline as anything else: clear interfaces between components, predictable behavior under edge cases, and a build system that actually works.

Technical Approach

All plugins are structured around Spigot's event system. Handlers register for specific server events, process them, and mutate game state accordingly. The key challenge in Manhunt was keeping compass bearings accurate at low latency across many simultaneous player movements.

// Compass update on PlayerMoveEvent @EventHandler public void onPlayerMove(PlayerMoveEvent e) { if (!isHunter(e.getPlayer())) return; Location target = speedrunner.getLocation(); updateCompassBearing(e.getPlayer(), target); }

Turtle Run introduced a different challenge: arbitrary bounding box intersection testing in 3D world space. Each checkpoint stores its AABB in world coordinates; every tick checks whether the runner's position falls inside any registered box, triggering checkpoint progression.

Build System

All plugins compile with Maven, using the Spigot API as a provided dependency. The build produces a standalone JAR dropped into a server's plugins directory, with no additional runtime dependencies. Plugin metadata is declared in plugin.yml, including version, main class, permissions, and command registrations.

What I Learned

These plugins were my first serious work with an event-driven architecture. Debugging race conditions in server-tick-driven state — where two events can fire in the same tick and leave state inconsistent — pushed me to think more carefully about ordering guarantees and idempotency. The Spigot API also exposed me to a real-world Java codebase with an enormous surface area, which taught me to navigate unfamiliar APIs efficiently.
