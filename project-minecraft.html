Minecraft Server Plugins

A collection of custom Spigot plugins built in Java for multiplayer Minecraft servers — from fully featured game modes to small quality-of-life utilities. Each plugin was written from scratch, compiled with Maven, and deployed on a live server for friends to play.

Java Spigot API Maven Event-Driven Multiplayer
Game Modes
Manhunt
Spigot 1.19.2 · Java
🏹

A full Minecraft Manhunt game mode where hunters track down runners across a freshly generated world. Supports multiple runners and hunters, full advancement resets, and cross-dimension portal tracking. Hunters receive a compass that tracks the nearest runner.

The compass updates continuously in real time via a PlayerMoveEvent listener — no right-clicking required. Implementing live lodestone metadata updates on every movement event without performance degradation required careful use of the Spigot scheduler and compass meta API.
Event Listener World Generation Compass API Multi-Dimension

Installation

Installing a Plugin on Your Server

01
Prerequisites — you need a running Spigot or Paper server. Download the server jar from papermc.io and run it once to generate the folder structure.
02
Get the .jar — each plugin's compiled .jar is in the target/ folder of its directory (e.g. Manhunt/target/Manhunt-1.0-SNAPSHOT.jar). You can also clone the repo and build with Maven: mvn clean package
03
Drop it in — copy the .jar into your server's plugins/ folder.
04
Restart — stop and restart your server. The plugin will load automatically on startup. You can verify it loaded with /plugins in the server console.
05
Run commands in-game — most plugins require operator permissions. Use /op <yourname> from the console, then run the plugin's command in-game (e.g. /manhunt <runner>, /timer add 10, /potionStart).
06
Spigot version — each plugin targets a specific Spigot version (listed in each card above). Using a mismatched server version may cause errors. Check the pom.xml in each plugin folder for the exact API version.
