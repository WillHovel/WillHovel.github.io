Overview

A custom-designed wearable PCB and firmware system for recognizing hand gestures from flex sensor data. Five flex sensors on the fingers produce analog resistance signals that vary with bend angle; these signals are read through the ESP32-S3's ADC channels, processed, and classified into distinct gestures using a trained classification algorithm.

The project spans the full hardware and software stack: schematic capture, PCB layout, component selection, firmware, data collection, and signal classification, all designed and built from scratch.

What I Built

  • Custom PCB (v2) — a circular board designed in KiCad, housing the ESP32-S3-Mini-1, a USB-C receptacle with proper CC resistors for power negotiation, an AP2112K 3.3V LDO regulator, decoupling capacitors, dual status LEDs, two pushbuttons (including a BOOT button for flashing), I2C and JTAG headers, and five analog input channels for the flex sensors
  • Flex sensor interface — each flex sensor is wired as a voltage divider with a 10kΩ pull-up resistor. The resulting analog voltage is read on a dedicated GPIO/ADC pin, giving a continuous bend angle signal per finger
  • Schematic and layout — full KiCad schematic with ERC-clean netlist, then routed PCB with antenna keepout, USB differential pair routing, and ground plane on both layers
  • Signal classification — collected training data across gesture classes and implemented a classification algorithm in Python to identify distinct hand poses from the five-dimensional sensor readings
  • Firmware — embedded code on the ESP32-S3 to sample the ADC channels, package sensor data, and handle USB communication for streaming and flashing

PCB Design

Assembled v2 board — circular form factor fits the back of the hand
KiCad layout showing antenna keepout, USB differential pair, and ground plane
Complete schematic: ESP32-S3, AP2112K LDO, USB-C power path, five flex sensor voltage dividers, LEDs, and programming buttons

The board is circular to fit cleanly on the back of the hand. The ESP32-S3-Mini-1 module sits centrally with its antenna region extending beyond the board edge, kept clear of copper pours. USB-C is handled by a 16-pin receptacle with the CC1/CC2 lines pulled to ground through 5.1kΩ resistors (standard for USB device mode). The AP2112K LDO regulates the USB 5V bus down to 3.3V for the ESP32 and sensor circuitry.

Five analog input headers break out to the flex sensor voltage dividers on the glove. Two LEDs provide power and status indication. The BOOT and reset buttons allow in-circuit firmware flashing without removing the board.

Signal Processing

Each flex sensor produces a noisy, slowly-varying analog signal. The core challenge is extracting reliable gesture classifications from five channels of correlated, non-stationary data, the same class of problem as extracting structure from biological motion signals. The approach involved collecting labeled training data for each target gesture, then training a classifier on the five-dimensional feature space defined by the per-finger bend readings.

This project introduced the central challenge that carries into my research work: how do you extract meaningful, repeatable structure from noisy sensor signals? Iterating on the sensor interface, data collection protocol, and classification algorithm until the system worked reliably taught me to think carefully about noise sources, feature representation, and the difference between a signal that looks clean and one that actually generalizes.
